Slide 1 / 2 — problema

Revisar PRs à mão é lento antes mesmo de ser difícil.

O revisor abre cada arquivo alterado, decifra o diff cru e só depois consegue julgar o que realmente importa. Grande parte desse tempo é triagem mecânica — e dá para automatizar.

Slide 2 / 2 — proposta

Um agente LangGraph entre o PR e o revisor.

Entrada owner/repo + nº do PR
→
fetch_pr valida acesso, busca metadata + diffs
analyze_files LLM avalia cada arquivo, acumula no estado
generate_report consolida achados em relatório final
handle_error PR inacessível ou inválido
→
Saída relatório .md estruturado

Ferramenta integrada

GitHub REST API (via requests), autenticada com token de leitura — busca real de metadata e diffs, não simulada.

Contexto / memória

O estado do grafo acumula os arquivos já analisados e o resumo parcial entre os nós, até o relatório final.

Stack

LangGraph Claude Haiku 4.5 (via OpenRouter) GitHub API Python
